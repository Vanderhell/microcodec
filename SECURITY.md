# Security Policy

## Supported versions

The latest release branch is the supported version for security fixes.

## Reporting a vulnerability

Please do not open public issues for security-sensitive bugs.

Instead, report the issue privately with:

- affected version or commit
- reproduction details
- impact assessment
- suggested mitigation if available

If a private channel is not yet configured in the hosting repository, create a
private contact process before publishing the project and link it from this
file.

## Scope

Security reports are especially welcome for:

- malformed input causing out-of-bounds reads or writes
- corruption handling failures
- issues affecting embedded or firmware workflows
- unintended state sharing across calls or threads
